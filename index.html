    • The waterfall model is one of the best known and oldest software development methods, named after the analogy of successive waterfalls - water falls from one level to the next and does not return..

    • This model is used only when the requirements are very well known, clear and fixed.
    • Product definition is stable.
    • Technology is understood.
    • There are no ambiguous requirements.
    • Ample resources with required expertise are available freely.
    • The project is short.

      • Advantages

      1. Before the next phase of development, each phase must be completed
      2. Suited for smaller projects where requirements are well defined

      Dis-advantages

      1. Error can be fixed only during the phase
      2. It is not desirable for complex project where requirement changes frequently

    Prototype-based programming is a style of object-based programming where behavior is reused through the process of reusing existing objects that act as prototypes. This model can also be called prototype, prototype-oriented, class-based, or instance-based programming.

    The spiral model is a risk-based software development process model. Based on the unique risk patterns of a given project, the spiral model directs the team to adopt elements of one or more process models, such as an incremental, waterfall, or evolutionary prototype.

    In software development, the V-model refers to a development process that can be considered an extension of the Kose model, and is an example of a more general V-model. Instead of moving linearly downwards, the process steps are bent upwards after the coding phase to form a typical V-shape.

    step 1: Define step 2: Design step 3: Build step 4: Test step 5: Release

    Field development is rapid software development that uses agile methodologies. According to this, project failure is considered to be the most costly aspect of software development, and therefore this type of software development aims to organize the work in a way that would increase the likelihood of success.

    Advantages

    1. You can deploy software quicker, so your customer can get value sooner rather than later
    2. You waste fewer resources because you always work on up-to-date tasks
    3. You can better adapt to change and respond faster Faster turnaround times
    4. You can detect and fix issues and defects faster
    5. You spend less time on bureaucracy and busywork
    6. There's a big community of Agile practitioners with whom you can share knowledge
    7. You can experiment and test ideas because its costs are low
      8. 7

    Dis-advantages

    1. Error can be fixed only during the phase
    2. Documentation tends to get sidetracked, which makes it harder for new members to get up to speed
    3. Agile demands more time and energy from everyone because developers and customers must constantly interact with each other
    4. When developers run out of work, they can't work on a different project since they'll be needed soon
    5. Features that are too big to fit into one or even several cycles are avoided because they don't fit in nicely into the philosophy
    6. Short cycles don't leave enough time for the design thinking process, so designers have to redevelop the experience over and over due to negative feedback
